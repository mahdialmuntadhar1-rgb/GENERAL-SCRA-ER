TROUBLESHOOTING QUICK FIXES
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Page shows no data"
FIX:     - Check Vercel env vars are set in Step 1
         - Refresh browser (Ctrl+F5 or Cmd+Shift+R)
         - Check browser console (F12) for errors

PROBLEM: "Page won't load (404 or 403)"
FIX:     - Wait 2-3 minutes, then refresh
         - Check Vercel Deployments → build logs
         - Verify env vars are correct in Step 1

PROBLEM: "See mock data instead of real data"
FIX:     - Check VITE_SUPABASE_URL is set
         - Check VITE_SUPABASE_ANON_KEY is set
         - Both must be in BOTH Production and Preview

PROBLEM: "Filtering doesn't work"
FIX:     - Check browser console (F12) for errors
         - Verify categories/locations exist in database
         - Try refreshing page

PROBLEM: "Build failed on Vercel"
FIX:     - Check Vercel build logs (Deployments tab)
         - This usually fixes itself with npm rebuild
         - Try redeploying from Vercel dashboard
